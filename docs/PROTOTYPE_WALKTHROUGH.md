## PART A — GYM MANAGEMENT (Admin Web Dashboard)

### 1. Centralized Operational Core

**SCRIPT**

"Let us first demonstrate the Admin Web Dashboard, which serves as the centralized operational core of CORE Fitness. The dashboard was developed using React.js and is designed to help gym administrators manage daily operations in one unified platform.

In traditional gyms, administrators usually rely on paper records, handwritten attendance logs, spreadsheets, and manual monitoring. Because of this, information becomes difficult to organize, easy to lose, and time-consuming to retrieve.

Through CORE Fitness, all important operational data are centralized digitally. Administrators can monitor member attendance, active memberships, revenue summaries, and payment activities in real time. The dashboard also provides quick statistical overviews that help administrators make faster and more informed decisions regarding gym operations."

### 3. QR-Based Attendance Monitoring

**SCRIPT**

"One of the key innovations of CORE Fitness is the QR-based attendance monitoring system.

Instead of using manual logbooks where members write their names manually, the system generates unique QR codes through the mobile application. Upon scanning the QR code, the system automatically validates the member's account and records attendance in real time.

This approach improves accuracy, minimizes manual work, prevents duplicate attendance entries, and allows administrators to monitor gym activity more efficiently."

## SYSTEM ARCHITECTURE INSERT

**SCRIPT**

"All these features are powered by an integrated technology stack composed of React.js for both web and mobile interfaces, Vite as the build tool, and localStorage for data persistence in this prototype version.

The web dashboard and mobile applications share a unified data layer through the SharedStorage utility, enabling real-time synchronization and cross-platform data consistency. The system uses React Router for navigation, Tailwind CSS for styling, and Framer Motion for smooth animations.

This architecture enables secure data management, real-time updates, centralized information storage, and efficient cross-platform operations. In a production environment, this would be extended with a Node.js/Express backend, MySQL database, and Firebase services for authentication and push notifications."

## KEY TALKING POINTS

### Problem Statement
- Manual record-keeping is inefficient and error-prone
- Paper-based attendance logs are difficult to track
- Payment monitoring lacks transparency
- Member communication is fragmented
- No data-driven insights for business decisions

### Solution Highlights
- **Centralized Management** — All operations in one platform
- **QR Attendance** — Automated, accurate, and secure
- **NLP Chatbot** — Reduces repetitive inquiries
- **Rule-Based Analytics** — Proactive member retention
- **RBAC** — Secure role-based access control
- **Cross-Platform** — Unified experience across web and mobile

### Technical Achievements
- React.js + TypeScript for type-safe development
- Responsive design with Tailwind CSS
- Smooth animations with Framer Motion
- Real-time data synchronization via SharedStorage
- Mobile-first design (375×812px frame)
- ISO 25010 quality compliance
